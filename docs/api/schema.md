# Database schema

Every table, view, and enum the dashboard reads or writes. Source of truth is the SQL in `supabase/migrations/` — this doc summarises shape, foreign keys, and RLS rules so you don't have to dig through 26 migrations.

`auth.users` is provided by Supabase Auth and not described here; what matters for the app is that `public.profiles.id = auth.users.id`.

### `stock_ledger` and `stock_on_hand`

Consumables are tracked by delta, not by serial.

`stock_ledger` columns:

| Column | Type | Notes |
|---|---|---|
| `id` | `uuid` PK | |
| `school_id` | `uuid` | |
| `product_id` | `uuid` | |
| `delta` | `int` | + on receipt, − on dispatch |
| `order_id` | `uuid` nullable | If from an assignment |
| `recorded_by` | `uuid` | FK → `profiles.id` |
| `recorded_at` | `timestamptz` | |
| `note` | `text` nullable | |

`stock_on_hand` is a view:

```sql
select school_id, product_id, sum(delta) as on_hand
from stock_ledger
group by school_id, product_id
```

## Views

### `public_schools_map`

Anonymous-readable subset of `schools` for the marketing-site map.

```
id, name, county, latitude, longitude, is_maker_space
```

### `stock_on_hand`

See above. School-scoped totals.

### `school_leaderboard`

Computed ranking. Pre-sorted DESC by `total_pts`.

```sql
school_id, school_name, county, programme_id, programme_name,
lesson_pts,           -- sum of stage.points across this school's
                      -- lesson_completions where passed = true
project_pts,          -- project_judgments.score (NULL → 0)
total_pts             -- lesson_pts + project_pts
```

**Grant** — `select` granted to `authenticated`. RLS still scopes the underlying tables; the view only exposes data the requester would already be allowed to count.
